最初の一歩

まずは、プログラムの最少単位の式をみてみよう。

式

  • プログラミングでは、実行して結果の得られるプログラムのことを式(expression)と呼ぶ。
  • 式を実行して実行結果を得ること「評価する」という。
Konohaでは、入力プロンプトに対し式を入力すると 
  >>>
  1+2
次のように評価され、値が得られる。 
  <<<
  3

数式

プログラミング言語は、もともと数学の記法を用いてプログラミングすることを設計方針としていた。 そのため、「可能な限り」数式をそのまま書ける。
  • 乗算は *
  • 除算は /
  • 余算は %
(演習) Konoha を電卓かわりに使って計算してみよう 
  >>>
  1+2*3

式 1+2*3を評価すると、評価値は7である。 この結果をみると、プログラミング言語でも、数式のルールにしたがい、 *演算子は+演算子よりも優先的に評価されていることがわかる。 つまり、1+(2*3)の順序である。

もし、加算を先に評価したいときは、(1+2)*3 のように優先する式を()で囲む。

(注意) プログラミング言語は、できるだけ「新しいことを覚えなくてよい」ように配慮して設計されている。 もし全く数式と違う記法だったら、そもそも使いにくくて仕方がないだろう。

変数

変数は、代数学において数値を抽象化して議論するために発明された道具である。 プログラミング言語は、「変数」を保持して参照する便利な道具として使っている。 次は、変数 x に 数値1を保持するため、割り当てる式である。 
  x = 1
変数 x に値を割り当てることを「代入」という。 変数は式である。変数を評価して、値を取り出すことを特に「参照」と呼ぶ。 (課題) x の値を参照してみよう 
  >>> x
通常、変数に改めて代入すると、変数の保持する値は再度割り当てられる。 次のように123を代入すると、x の値は123に変更される。 
>>>
x = 123
x
<<<
123

再帰的な文法定義

プログラミング言語の文法の重要な原則を学んでおこう。

まず、今まで習ってきた式を思い出してみよう。

式 は、いずれかのどれかである。

  • 数値
  • 変数
  • ＜式＞ + ＜式＞
  • ＜式＞ * ＜式＞
  • (＜式＞)

このように、式の定義に式が含まれる構造を再帰的構造と呼ぶ。

プログラミング言語の文法は、どのような言語であっても再帰的な構造で定義されている。 再帰構造を用いることで組み合わせが自由になり、式を組み合わせてより複雑な式を 表現することができるようになる。

手続きと宣言

最後に、プログラミングが数学と異なる重要な違いを理解してみよう。

まず、次の式をみてみよう。 

  x = x + 1

もし数学的に解釈するのなら、x = x + 1 という式は矛盾しており、成り立たない。

プログラミング言語では、「変数xに x+1を評価した値を代入する」という意味になる。

プログラミング言語は、手続きを記述するものである。 手続きによって状態を変更することができる。

逆に、数学のように命題を宣言するものではない。
